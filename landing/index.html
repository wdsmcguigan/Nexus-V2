Features

THREE THINGS NEXUS GETS RIGHT

Most clients get one of these. Nexus ships all three — together, from day one.

A Schema You Control

Unlimited custom fields per message. Workflow statuses from Triage to Done. Priority, stars, flags, tags, and markdown notes — all indexed and filterable. Not metadata tags. An actual schema you designed.

  • Custom fields: text, number, date, select, person, URL, boolean
  • 12-style star system (stars, hearts, bookmarks, flags)
  • Kanban workflows: drag messages through status columns
  • P0–P3 priority, mute, pin, flag with due date + reminder
  • Markdown notes per message, full-text searchable
  • Multi-axis saved views — one-click access to complex filters
Your Data, Actually Yours

SQLCipher-encrypted vault on your disk. Maildir .eml files you can see in Finder. Optional E2EE relay sync that the server — including ours — cannot decrypt. Exit anytime; your mail stays readable.

  • SQLCipher (AES-256) encrypted SQLite vault
  • Maildir .eml files — visible in Finder, portable to any client
  • XChaCha20-Poly1305 E2EE relay — zero-knowledge multi-device sync
  • Device enrollment via 6-digit code + BLAKE3 key derivation
  • Argon2id password KDF — no brute-force weak-key attacks
  • No annotations ever leave for Google or Apple servers
Built Like Your IDE

⌘K command palette. Dockable resizable panels. Per-workspace layouts. Sub-10ms full-text search across 100k+ messages. Everything reachable by keyboard.

  • ⌘K command palette — every action, zero mouse required
  • Dockable panels: list, inspector, composer — save per workspace
  • List / Kanban / Table view toggles per panel
  • FTS5: subject + body + notes searched together, offline
  • Rules engine: auto-label, auto-status, auto-assign
  • Tauri 2 native shell — no Electron, ~10MB binary

The Researcher's Vault

Every newsletter, paper, and link — actually searchable.

Tag threads by topic. Add a Read boolean and a Notes markdown field. FTS5 searches subject, body, and notes in one query. Filter Tag = ML AND Read = false for your reading list. No clipping, no pasting, no subscription fee.

Why Nexus, Why Now

THE GAP IN THE MARKET IS REAL

An independent strategic analysis found no existing client combining all four of Nexus's core bets. This is the right architectural bet at the right time.

01 The only client combining local-first + E2EE sync + metadata layer. Pebble has local storage but no sync. Velo has local storage but no metadata. Superhuman has speed but no privacy. Nexus is the first to ship all three together.
02 Maildir on disk means zero lock-in — a trust signal no cloud client can match. Your .eml files are readable by any IMAP client tomorrow, whether Nexus exists or not.
03 The JMAP roadmap positions Nexus as the definitive power-user client for self-hosted mail. Stalwart, Mox, and Fastmail all speak JMAP. As self-hosting grows, Nexus becomes the obvious frontend.
04 Open source means the moat is auditable. The E2EE relay is open. The vault schema is open. Contributors can verify the privacy claims and extend them.
