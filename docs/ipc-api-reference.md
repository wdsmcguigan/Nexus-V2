# IPC API Reference

Canonical reference for all **56 Tauri IPC commands** exposed by the Rust backend, generated by reading `src-tauri/src/commands.rs` and `src/storage/tauri.ts` directly. Grouped by feature area, matching the comment blocks in `src-tauri/src/lib.rs:invoke_handler!`.

> **Pattern:** every command needs three things — (1) `#[tauri::command] pub async fn` in `commands.rs`, (2) entry in `invoke_handler![…]` in `lib.rs`, (3) typed wrapper in `src/storage/tauri.ts`. If any one is missing, the command is not callable.
>
> **Source of truth:** if this doc disagrees with the code, the code wins. Last verified 2026-05-28.

## Calendar (EP-10 / EP-11)

> Calendar events are **hydrated** into the store by `load_vault_data`. There is intentionally no `get_calendar_events` IPC. Deletion goes through the `DELETE_CALENDAR_EVENT` mutation, not a dedicated command.

| Command | TS wrapper | Returns | Notes |
|---|---|---|---|
| `sync_google_calendar(accountId)` | `syncGoogleCalendar(accountId)` | `number` (count) | Pulls all selected calendars (multi-calendar). |
| `create_calendar_event({...})` | `createCalendarEvent({...})` | `string` (event ID) | Inserts locally + pushes to Google. |
| `update_calendar_event({...})` | `updateCalendarEvent({...})` | `void` | **Identifies events by `externalId`, not internal `id`** — using `id` here silently fails to push to Google. |
| `get_calendar_list(accountId)` | `getCalendarList(accountId)` | `CalendarListEntry[]` | Multi-calendar selector data. |
| `search_calendar_events(query, vaultId, limit)` | `searchCalendarEvents(query, vaultId, limit=50)` | `string[]` (event IDs) | FTS5 over `calendar_events_fts`. |

## How to add a new command

1. **Implement** the function in `src-tauri/src/commands.rs`, tagged with `#[tauri::command] pub async fn`. Return `std::result::Result<T, String>` (Tauri serializes the error string).
2. **Register** it in `src-tauri/src/lib.rs:invoke_handler!` under the appropriate epic comment.
3. **Wrap** it in `src/storage/tauri.ts` with a typed wrapper (`camelCase` name; pass args as a single object so Tauri's `serde` keyword-to-snake-case mapping works).
4. If the command **mutates state**, also add:
   - A `MutationKind` variant in `src/data/types.ts`
   - A case in `src/state/mutations.ts:applyMutation()` (optimistic frontend update)
   - A helper function in `src/state/mutations.ts` (`recordMutation("KIND", payload, store)`)
   - Backend persistence in `src-tauri/src/db/queries.rs`
5. Update this file.
